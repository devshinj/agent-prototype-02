# 태스크 캘린더 개선 설계 스펙

## 개요

태스크 캘린더에 저장소별 필터링과 기간 선택 기능을 추가하여, 선택한 저장소/기간에 대한 보고서를 생성할 수 있도록 한다.

## 현재 상태

- 1/3개월 캘린더 뷰, 날짜별 커밋 히트맵
- 날짜 클릭 시 전체 저장소의 커밋 상세 표시
- 단일 날짜 + 단일 저장소 기준 AI 보고서 생성
- 저장소 필터링 없음, 기간 선택 불가

## 변경 사항

### 1. 캘린더 상단 — 저장소 필터 + 기간 프리셋

#### 저장소 멀티셀렉트

- Popover + Command(검색 가능) 조합
- 접힌 상태: 선택된 저장소명을 태그/칩으로 표시, 3개 초과 시 "+N more"
- "전체 선택/해제" 토글
- 선택 변경 시 캘린더 히트맵 즉시 갱신 (선택된 저장소의 커밋만 집계)

#### 기간 프리셋 바

- 버튼 그룹: `이번 주` | `지난 주` | `이번 달` | `지난 달` | `커스텀`
- `커스텀` 클릭 시 캘린더에서 시작일/종료일 두 번 클릭으로 범위 지정
- 프리셋 선택 시 캘린더에서 해당 기간이 하이라이트됨
- 단일 날짜 클릭은 기존처럼 동작 (프리셋 해제)

### 2. 캘린더 하단 패널 — 날짜/기간별 상세 + 보고서

#### 단일 날짜 선택 시 (기존과 동일)

- 저장소 > 브랜치 > 커밋 아코디언
- 필터링된 저장소만 표시
- 각 저장소 카드에 "보고서 작성" 버튼

#### 기간 선택 시

- 날짜 > 저장소 > 브랜치 > 커밋 아코디언 (날짜 그룹 한 단계 추가)
- 필터링된 저장소만 표시
- 상단에 기간 요약 (총 커밋 수, 활동 일수, 저장소별 커밋 수)
- 각 저장소별 "기간 보고서 작성" 버튼

#### 보고서 생성 (비동기)

- 버튼 클릭 → 확인 다이얼로그 → 즉시 "생성 요청됨" 토스트 표시
- 백그라운드에서 Gemini가 보고서 생성
- DB에 `status: "pending" | "completed" | "error"` 상태 관리
- 보고서 목록(`/reports`)에서 생성 완료 여부 확인 가능
- 캘린더 페이지에서도 해당 저장소에 "보고서 생성 중..." 인디케이터 표시

### 3. API 변경 사항

#### 기존 API 수정

**`GET /api/repos/commit-calendar`**
- 파라미터 추가: `repoIds` (쉼표 구분, 선택적 — 없으면 전체)
- 선택된 저장소만 커밋 카운트 집계

**`GET /api/repos/commit-calendar/[date]`**
- 파라미터 추가: `repoIds` (동일)

#### 신규 API

**`GET /api/repos/commit-calendar/range`**
- 파라미터: `since`, `until`, `repoIds`
- 응답: 기간 내 날짜별 > 저장소별 > 브랜치별 커밋 상세
- 기존 `[date]` API의 기간 확장 버전

**`POST /api/reports/generate`**
- 기존 body에 `dateRange?: { since: string; until: string }` 추가
- `date`만 있으면 단일 날짜 보고서 (기존 호환)
- `dateRange`가 있으면 기간 보고서 Gemini 프롬프트 사용

#### DB 변경

**`reports` 테이블:**
- `date_start`, `date_end` 컬럼 추가 (기간 보고서용)
- 기존 `date` 컬럼은 단일 날짜 보고서용으로 유지
- `status` 컬럼 추가: `"pending" | "completed" | "error"` (기본값 `"completed"` — 기존 보고서 호환)

### 4. 컴포넌트 분리 구조

현재 `task-calendar/page.tsx`가 모든 로직을 담고 있으므로 다음과 같이 분리:

```
src/app/(dashboard)/task-calendar/
├── page.tsx                    # 페이지 컨테이너 — 상태 관리 + 조합
├── components/
│   ├── repo-filter.tsx         # 저장소 멀티셀렉트 (Popover + Command)
│   ├── period-presets.tsx      # 기간 프리셋 버튼 그룹 + 커스텀 범위 상태
│   ├── calendar-grid.tsx       # 월 캘린더 히트맵 (기존 MonthGrid 추출)
│   ├── date-detail-panel.tsx   # 단일 날짜 상세 (저장소 > 브랜치 > 커밋)
│   └── range-detail-panel.tsx  # 기간 상세 (날짜 > 저장소 > 브랜치 > 커밋)
```

- `page.tsx`는 필터/기간/날짜 상태를 관리하고 하위 컴포넌트에 전달
- 단일 날짜 vs 기간에 따라 `date-detail-panel` / `range-detail-panel` 조건 렌더링
- 보고서 생성 로직은 패널 컴포넌트 내에서 처리

## 접근 방식

기존 `/task-calendar` 페이지를 확장하는 방식 (방식 1). 기존 흐름을 유지하면서 필터와 기간 선택을 추가하고, 컴포넌트를 적절히 분리하여 유지보수성 확보.

## 범위 외

- 실시간 보고서 생성 진행률 표시 (폴링으로 상태 확인하는 것으로 충분)
- 보고서 자동 생성 스케줄링
- 여러 저장소를 합친 통합 보고서 (저장소별 개별 보고서만)
